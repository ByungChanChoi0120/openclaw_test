# WORKFLOW_AUTO.md

## 목적
실행 속도는 올리고, 품질 저하는 막는다.

## 위임 원칙 (Sub-agent / sessions_spawn)

### 1) 위임해도 되는 일 (판단 최소화)
- 코드 수정(파일/조건/결과가 명확)
- 데이터 조회/집계
- 포맷 변환, 파일 정리
- API 호출/배치 실행

### 2) 직접 해야 하는 일 (맥락/취향/전략)
- 브랜딩/디자인 톤 결정
- 문장 톤/콘텐츠 기획
- 우선순위/전략 판단
- 대외 커뮤니케이션 초안 최종본

### 3) Task 작성 규칙 (필수)
아래 6가지를 task에 반드시 넣는다.
1. 대상: 파일/경로/리소스
2. 목표: 완료 기준(Definition of Done)
3. 제약: 절대 지켜야 할 규칙
4. 참고: 읽어야 할 문서 경로
5. 검증: 테스트/빌드/체크 방법
6. 산출물: 결과물 형식

### 4) 금지
- "대충 만들어줘" 같은 모호 지시
- GUI 자동화(카톡/데스크톱 앱 등) 동시 위임
- 스타일/브랜드 판단 작업의 무감독 위임

### 5) 기본 템플릿
```js
sessions_spawn({
  task: `
[대상]
- 수정 파일: ...

[목표]
- ... 상태를 만족할 것

[제약]
- ... 절대 준수

[참고]
- read: ...

[검증]
- npm run build 통과
- 테스트 ... 통과

[산출물]
- 변경 요약 3줄
- 수정 파일 목록
- 리스크/후속 TODO
`,
  label: "작업명",
  runTimeoutSeconds: 600
})
```

### 6) 품질 게이트
- 실패 시 재시도 전에 task를 먼저 재작성
- 결과가 애매하면 본체가 최종 판단
- 반복 실수는 AGENTS.md에 규칙으로 승격
